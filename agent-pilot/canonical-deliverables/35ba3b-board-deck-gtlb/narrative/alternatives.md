# Alternative Narrative Structures Considered and Rejected

## Structure 1: Chronological Research Process
**Description:** Data collection → Analysis → Valuation → Conclusion
**Rejected because:** The board has already read the memo. A chronological approach would feel like a lecture, not a discussion. The board wants to evaluate the recommendation, not watch the agent work.

## Structure 2: Risk-First
**Description:** Risks → Mitigations → Thesis → Recommendation
**Rejected because:** This frames the entire investment as defensive. It signals "I'm worried about this" rather than "I'm excited about this." For a BUY recommendation, we want to lead with the opportunity.

## Structure 3: Competitive Analysis First
**Description:** Market landscape → Competitive position → Financials → Valuation
**Rejected because:** The competitive context is secondary to the financial trajectory. The board needs to understand the numbers before they can evaluate the competitive positioning. Also, the competitive analysis is more interesting when it serves as evidence for the thesis, not as the thesis itself.

## Structure 4: Valuation First
**Description:** DCF → Comps → Scenarios → Thesis → Recommendation
**Rejected because:** The valuation is only meaningful in the context of the thesis. A DCF number without the underlying assumptions (revenue growth, margin expansion) is just a number. The board needs to understand the story before the math.

## Structure 5: Three-Act (Setup → Confrontation → Resolution)
**Description:** Current state (setup) → Why the market is wrong (confrontation) → Why we're right (resolution)
**Rejected because:** This is too dramatic for a board presentation. It works for a pitch deck but not for a technical audience that values substance over narrative flair.

## Structure 6: Question-Driven
**Description:** Frame each section as answering a specific question ("Is GitLab growing?" "Is it profitable?" "Is it undervalued?")
**Rejected because:** While intellectually elegant, this structure makes the deck feel like a Q&A session rather than a coherent argument. The board wants a narrative, not a checklist.

## Structure 7: Investment Committee Format
**Description:** Formal IC memo format: Executive Summary → Investment Thesis → Financial Analysis → Risk → Recommendation
**Rejected because:** This is too close to the memo itself. The deck should be a discussion tool, not a document re-delivery. The board has the memo; they need the deck to facilitate conversation.

## Final Structure: Recommendation → Evidence → Valuation → Process → Verification
**Why this works:**
1. **Leads with the answer** — Board knows the recommendation immediately
2. **Shows the evidence** — Financial trajectory, competitive position, mispricing thesis
3. **Explains the valuation** — Bear/base/bull, DCF bridge
4. **Demonstrates the process** — Reasoning trail, dead ends (capability demo)
5. **Enables verification** — Audit mechanism, reconciliation (trust builder)

This structure serves two masters simultaneously: it presents the investment case (for the growth investor and value investor) AND demonstrates the agent's reasoning capability (for the technical skeptic and risk manager).
